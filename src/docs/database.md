---

# SKU Generation

SKU is NOT generated by the application.

SKU is generated by PostgreSQL.

Implementation

- PostgreSQL Sequence
- BEFORE INSERT Trigger

Format

JL-000001

JL-000002

JL-000003

Rules

- Generated only when SKU is NULL.
- Never overwrite existing SKU.
- Concurrency Safe.
- Unique.

# Stock Transactions

Table

stock_transactions

Columns

| Column | Type | Description |
|----------|----------|----------------------------|
| id | uuid | Primary Key |
| product_id | uuid | Related Product |
| transaction_type | text | GOODS_RECEIPT / STOCK_OUT / ADJUSTMENT |
| quantity | integer | Positive Quantity |
| reference_number | text | Optional Reference |
| remarks | text | Optional Remarks |
| performed_by | text | User Name |
| created_at | timestamptz | Transaction Time |

---

# Transaction Types

GOODS_RECEIPT

Adds stock.

STOCK_OUT

Removes stock.

ADJUSTMENT

Sets stock to the physical counted quantity.

---

# Inventory Engine

Inventory updates are NOT performed directly.

All stock movement must go through

apply_stock_transaction()

The RPC performs

1.

Lock Product Row

↓

2.

Validate Transaction

↓

3.

Update Product Stock

↓

4.

Insert Transaction

↓

5.

Return Transaction

Everything happens atomically.

# Business Rules

## Products

---

---

---

Quantity must always be positive.

Transaction Type determines whether stock is increased or decreased.
